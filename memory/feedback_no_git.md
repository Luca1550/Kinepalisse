Ne pas lancer de commandes git (git diff, git status, git log, git commit, git push…). L'utilisateur s'en occupe entièrement.

**Why:** Préférence explicite de l'utilisateur.

**How to apply:** Toujours sauter les étapes git dans le workflow. Ne pas proposer de commit ni de diff même pour vérifier les changements.
